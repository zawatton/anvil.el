* What is Anvil?

Anvil exposes Emacs capabilities as [[https://modelcontextprotocol.io/][Model Context Protocol (MCP)]] tools
that any AI coding assistant can call — Claude Code, Claude Desktop,
GPT, local models via Ollama/llama.cpp, or anything that speaks MCP.

One ~(anvil-enable)~ and your Emacs becomes an AI-accessible workbench
with 50+ tools: file editing, org-mode operations, git inspection,
Elisp evaluation, and more — all through a safe, sandboxed interface.

* Why Anvil?

- *LLM-agnostic* — works with any MCP-compatible client
- *Safe by design* — worker daemon pool isolates AI tool calls from your editor
- *Org-mode native* — first-class org read/write/refactor support
- *Large file safe* — handles 1.2MB+ files without data loss or truncation
- *Cross-platform* — Linux, macOS, Windows, WSL
- *Modular* — load only what you need, extend with your own modules

* Modules

| Module       | Tools                                          | External Deps    |
|--------------+------------------------------------------------+------------------|
| =worker=     | Isolated sub-Emacs daemon pool for AI calls    | —                |
| =eval=       | Sync/async Elisp evaluation                    | —                |
| =org=        | Read/write org headings, outlines, TODO states | org-mode         |
| =file=       | Safe file ops, batch edits, replace/regexp     | —                |
| =host=       | CPU, RAM, disk, GPU, network, OS info          | —                |
| =git=        | Status, log, diff-stats with CJK path safety  | git              |
| =proc=       | Process listing and inspection                 | —                |
| =net=        | Port and network inspection                    | —                |
| =fs=         | Directory trees, disk usage, recent files      | —                |
| =emacs=      | Buffer state and disk sync diagnosis           | —                |
| =text=       | URL extraction, link parsing                   | —                |
| =clipboard=  | OS clipboard read/write                        | —                |
| =data=       | JSON/CSV read/write with UTF-8 enforcement     | —                |
| =xlsx=       | Excel read/write (optional)                    | Python, openpyxl |
| =pdf=        | PDF text extraction (optional)                 | Python, pymupdf  |
| =ide=        | xref, diagnostics, imenu, tree-sitter (opt.)   | project.el       |
| =cron=       | Scheduled task runner with worker dispatch      | —                |
| =elisp=      | Agentic Elisp development tools (optional)     | —                |

* Quick Start

#+begin_src emacs-lisp
(require 'anvil)
(anvil-enable)
(anvil-server-start)
#+end_src

Then run =M-x anvil-describe-setup= for client configuration instructions.

* Installation

** async-installer

#+begin_src emacs-lisp
(async-installer-github-add "https://github.com/zawatton21/anvil.el.git"
                            :branch "master"
                            :main "anvil.el")
#+end_src

** straight.el

#+begin_src emacs-lisp
(straight-use-package
 '(anvil :type git :host github :repo "zawatton21/anvil.el"))
#+end_src

** elpaca

#+begin_src emacs-lisp
(elpaca (anvil :host github :repo "zawatton21/anvil.el"))
#+end_src

** use-package (with straight or elpaca)

#+begin_src emacs-lisp
;; with straight
(use-package anvil
  :straight (anvil :type git :host github :repo "zawatton21/anvil.el"))

;; with elpaca
(use-package anvil
  :ensure (:host github :repo "zawatton21/anvil.el"))
#+end_src

** Manual

Clone this repository and add it to your =load-path=:

#+begin_src emacs-lisp
(add-to-list 'load-path "/path/to/anvil")
(require 'anvil)
#+end_src

* Configuration

#+begin_src emacs-lisp
;; Choose which modules to load (defaults shown)
(setq anvil-modules '(worker eval org file host git proc fs emacs text clipboard data net))

;; Enable optional modules
(setq anvil-optional-modules '(xlsx pdf ide cron))

;; Start anvil
(anvil-enable)
(anvil-server-start)
#+end_src

* Architecture

#+begin_example
  Claude / GPT / Local LLM
         |
    MCP (JSON-RPC via stdio)
         |
  emacs-mcp-stdio.sh (bridge)
         |
    emacsclient → Emacs daemon
         |
    anvil-server (dispatcher)
         |
    +---------+---------+
    | main    | worker  |  (worker pool: heavy ops
    | daemon  | daemons |   run in isolated sub-Emacs)
    +---------+---------+
#+end_example
