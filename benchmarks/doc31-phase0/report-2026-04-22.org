** Fix during G2

Initial version mis-classified char-table as ~other~ because
~(vectorp ct)~ returns ~nil~ for char-tables.  Moved
~(char-table-p v)~ branch before ~(vectorp v)~.  Now correctly reports
~char-table~ type and samples ASCII chars.

** Format recommendation

S-exp is consistently ~10-25% smaller~ than JSON across all types.  On a
~90-tool MCP server where every response eats token budget, that is
non-trivial.

Proposed resolution: support **both** formats via a ~:format= keyword on
~anvil-inspect-object= (default = ~sexp~ for token efficiency, ~json~
requested explicitly when downstream ACP editor / automation needs
stricter grammar).
