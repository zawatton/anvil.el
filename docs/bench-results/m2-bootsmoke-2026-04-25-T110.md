## Subprocess support: PASS

`scripts/m2-bootsmoke.sh` wires the full subprocess substrate stack on top of the M1 base:

```
;; substrate-features:    load=t ec=t ec-fileio=t rx=t json=t sqlite=t coding=t b64=t hash=t ec-bridge=t
;; subprocess-features:   eventloop=t actor=t mux=t process=t
;; anvil-features:        anvil=t server=t commands=t defs=t state=t host=t worker=t
;; subprocess-spawn-probe: exit=0
;; multiplex-tick-probe:  fired=0 backend=host-bridge
```

Concrete claims verified:

1. `nelisp-process` (T106 SHIPPED, Phase 9d.L) loads cleanly under the host harness and `nelisp-make-process` spawns a `sh -c` child whose exit code (0) round-trips through `nelisp-process-wait-for-exit` within 50 ms.
2. `nelisp-eventloop-multiplex` (T103 SHIPPED, Phase 9d.K) reports backend=`host-bridge` and `nelisp-eventloop-multiplex-tick 50` returns an integer (0 fired, empty registry) without raising.
3. `nelisp-actor` + `nelisp-eventloop` (transitive deps of `nelisp-process`) load without dependency churn.
4. The Phase 9d.L filter trampoline mirrors child stdout into the wrap buffer (`m2-bootsmoke-test-nelisp-process-captures-stdout`), matching the contract anvil-host's `anvil-shell` depends on.
