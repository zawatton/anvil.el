** Audit corrections from Phase B + D-E execution

Phase A の audit には 3 件の精度補正:

- §2.1 *anvil-buffer*: "drop-in compatible" 表現は不正確。NeLisp の
  =nelisp-emacs-compat.el= は drop-in 置換でなく、=nelisp-ec-*= prefix
  の **namespace-disjoint な parallel buffer world**。Stage D = NeLisp
  が Emacs *内*で動く構成、anvil-buffer はネイティブ Emacs buffer 使用、
  両 buffer world が共存。PURE 分類は実用上正解だが理由が違う。

  実は **docstring 内 backquote ref** (= =anvil-browser.el:119=)、

  Phase B5 では docstring を明確化 (= 4 行修正) のみ実施。

* §0 要旨

NeLisp standalone (= Doc 32 v2 Phase 7.5) 完成見込みと整合させて、
=anvil.el= を「AI 用 workbench (= NeLisp 上で動く、Emacs 不要)」と
「人間用 IDE 拡張 (= Emacs 必要)」の 2 layer に分離する。

** §0.1 戦略 message

- =anvil.el= 名は維持 (= brand)、AI 用 main 主力
- =anvil-ide.el= を新規切り出し、Emacs 必須機能のみ収納
- 依存方向: =anvil-ide.el= → =anvil.el= (= IDE は tools の上に乗る)
- 配布: =anvil.el= は NeLisp standalone tarball 同梱、=anvil-ide.el= は
  Emacs ユーザー向け MELPA-style 別 install

** §0.2 期待効果

| 項目 | 現状 | 切り出し後 |
|------+------+------------|
| AI 利用 checkout | anvil.el 60+ module (~150K LOC) | anvil.el ~57 module (~144K LOC、Emacs 不要) |
| Emacs install 要否 | 必須 (= 隠蔽 wrapper でも) | 不要 (= NeLisp standalone で完結) |
| 依存明示 | 60 module 全部相互依存 | tools / ide 2 layer + 単方向依存 |
| repo 分離 | mono | =zawatton/anvil.el= + =zawatton/anvil-ide.el= |

** §1.1 API surface 確認結果

| API surface | NeLisp side | status |
|-------------+-------------+--------|
| byte-compile | =nelisp-cc-runtime-compile-and-allocate= (form-based) | partial (= file-based なし、form-loop で代替) |

| tree-sitter | none | absent → ide 行き |
| find-file-noselect | =nelisp-ec-insert-file-contents= 2-step | partial (= 2-step adapter で代替可) |

** §1.2 結論

- **anvil-buffer**: 既に PURE (= adapter 不要、NeLisp で drop-in 動作)
- **anvil-elisp**: 6/7 tool 移植可、=elisp-info-lookup-symbol= 1 件のみ ide
- **anvil-org-index**: 直接 delegate で PURE 化可
- **anvil-org**: 大半 PURE、=org-edit-body= 等 tree-walk 系のみ ide
- **anvil-treesit / anvil-ide / anvil-worker-ui**: 真の ide
