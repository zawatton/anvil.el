* Motivation

vibecode-pro-max-kit は RIPER-5+2 の 7 段ゲート
(RESEARCH / SPEC / INNOVATE / PLAN / VALIDATE / EXECUTE /
UPDATE-PROCESS) と PVL/EVL ループ (最大 10 サイクル) を謳う。実体は
「plan-before-act」の再パッケージで、anvil-orchestrator
(claude=設計 / codex=実装) + openspec-pilot + plan-mode で思想は
既に網羅済み。「自己改善コンテキストメモリ」も md 手編集に過ぎず、
anvil の DB-primary memory が厳密に上位互換。

ただし 2 点、anvil orchestrator に *明示的に欠けている gate* がある:

1. *リトライ上限* — PVL/EVL は最大 10 サイクルと数値で打ち切る。
   anvil-orchestrator の検証/再投げに明示的な収束上限が無い。
2. *INNOVATE ゲート* — PLAN 前に「2–3 案を出して trade-off 比較」を
   必須化している。anvil-orchestrator には PLAN 前の「案比較」ゲートが
   無い (consensus preset は実装案の合議だが、設計の代替案列挙とは別)。

* Goals
1. orchestrator のループ系 tool に *retry / cycle 上限* を defcustom で
   導入し、超過時は fail-loud で停止 + レポート (無限ループ防止)。
2. *INNOVATE preset* — PLAN 前に「N 案 + trade-off 表」を 1 ステップ
   挟む orchestrator preset / 規約。Doc 19 (peer review) /
   Doc 23 (consensus preset) の延長で実装。

* Non-goals
- vibecode の =.claude/agents,skills,hooks= / install スクリプト /
  context-group Markdown の移植 — *一切しない*。anvil の DB / MCP が
  上位互換。
- 7 段ゲートの full 採用 — anvil の orchestrator role-split + openspec
  で足りる。採るのは上記 2 概念のみ。
