* Goal

claude-mem が示した 5 つの設計パターンを anvil-memory に取り込み、

1. 5 hook 自動キャプチャによる observation 蓄積
2. AI 圧縮による long-term context 化
3. 3-layer progressive disclosure search (~10x token saving)
4. session 横断の context 自動 inject
5. 重複・矛盾検出を含む memory health 機能

を anvil-memory のみで完結させる。

** Non-goals
- claude-mem 本体の install / 利用 (AGPL 汚染を完全回避)
- claude-mem コードの直接コピー (アイデア参照のみ、実装は独自)
- HTTP server (port 37777 相当) 提供 (Emacs in-process で完結)
- Chroma vector DB のような Python 依存追加

* SQLite schema

#+begin_src sql
CREATE TABLE sessions (
  id            TEXT PRIMARY KEY,            -- claude session id
  started_at    INTEGER NOT NULL,            -- unix epoch
  ended_at      INTEGER,
  project_dir   TEXT,
  meta_json     TEXT
);

CREATE TABLE observations (
  id            INTEGER PRIMARY KEY AUTOINCREMENT,
  session_id    TEXT NOT NULL REFERENCES sessions(id),
  ts            INTEGER NOT NULL,
  hook          TEXT NOT NULL,               -- session-start / user-prompt / post-tool-use / stop / session-end
  tool_name     TEXT,                         -- post-tool-use 限定
  payload_json  TEXT,                         -- raw event data
  body          TEXT,                         -- 検索対象 (tool 名 + args + result の textual form)
  importance    INTEGER DEFAULT 0,            -- 0..100, AI 圧縮時に更新
  is_compressed INTEGER DEFAULT 0
);

CREATE TABLE summaries (
  id            INTEGER PRIMARY KEY AUTOINCREMENT,
  session_id    TEXT NOT NULL REFERENCES sessions(id),
  obs_start_id  INTEGER,                      -- 圧縮した observation 範囲
  obs_end_id    INTEGER,
  topic         TEXT,                         -- AI 抽出トピック
  summary       TEXT NOT NULL,                -- 1-3 sentence semantic summary
  ts            INTEGER NOT NULL
);

CREATE VIRTUAL TABLE observations_fts USING fts5(
  body, content='observations', content_rowid='id',
  tokenize='trigram'                          -- CJK 対応
);

CREATE VIRTUAL TABLE summaries_fts USING fts5(
  topic, summary, content='summaries', content_rowid='id',
  tokenize='trigram'
);
#+end_src

* Phase plan

** Phase 1 — Observation capture (SHIPPED 2026-04-25, merge 37fab6b)
*** 1.1 hook handler 拡張
- =anvil-hook.el= の =anvil-hook-dispatch= で各 event 後に
  =(anvil-memory-observation-record HOOK SESSION-ID PAYLOAD)= を呼ぶ
- raw payload は =payload_json= に prin1、検索用 =body= は textual extraction
*** 1.2 schema migration
- =anvil-memory--ensure-schema= に observations / summaries / sessions 追加
- 既存の memory_index テーブルとは独立 (auto-memory MD は従来通り)
*** 1.3 importance 推定 (rule-based)
- error / fix / commit 系 keyword: +30
- 大きい diff (>100 lines): +20
- repeated tool call (同 tool 連続): -10
- 後続で AI 圧縮時に再評価

** Phase 2 — AI compression worker (SHIPPED 2026-04-25, merge 6071b92)
*** 2.1 SessionEnd 時に圧縮
- =anvil-orchestrator-submit= で claude provider に
  「直近 N 件 observation を 1-3 sentence で要約 + topic 抽出」
- result を summaries テーブル insert
*** 2.2 token cost guard
- =defcustom anvil-memory-compress-monthly-budget-tokens= (default 100k)
- 月予算超過時は圧縮 skip + log warning
*** 2.3 失敗時 fallback
- claude API 不通 → rule-based 要約 (最初の200char + last200char で代替)

** Phase 3 — 3-layer search MCP tools (SHIPPED 2026-04-25, merge 244eda1)
*** 3.1 =memory-obs-search= (Layer 1: index)
- input: =query=, =limit= (default 10), =filter= (session/date/hook)
- output: =[(:id N :ts T :hook H :preview "..." :score S)]=  (~50-100 tokens/件)
- FTS5 BM25 ranking
*** 3.2 =memory-obs-timeline= (Layer 2: context)
- input: =anchor-id=, =window= (default ±5)
- output: anchor の前後 observation を時系列で
*** 3.3 =memory-obs-get= (Layer 3: full body)
- input: =ids= (array), max 20
- output: =[(:id N :body "..." :payload {...})]=
*** 3.4 =memory-summary-search= (高速トピック検索)
- summaries_fts 直接、~30 tokens/件
- 「過去にこの問題どう解いたか」用

** Phase 4 — Auto-inject on SessionStart (SHIPPED 2026-04-25, merge 089b32c)
*** 4.1 関連 summary の prepend
- =SessionStart= hook 発火時、=current project-dir= を key に
  直近 7 日の summaries を関連度 top-5 抽出
- =CLAUDE_INJECT_CONTEXT= 環境変数 or hook output で context 注入
*** 4.2 token budget
- 注入は max 1000 tokens、優先度 = recency × importance × project_match
*** 4.3 opt-out
- =defcustom anvil-memory-auto-inject= (default nil 安全側)

** Phase 5 — UI (Tabulated list, SHIPPED 2026-04-25, merge 3f43492)
*** 5.1 =M-x anvil-memory-observations=
- session ごとに observation を table 表示
- column: ts / hook / tool / preview / importance
- =RET= で full body 表示、=p= で promote、=d= で delete
*** 5.2 =M-x anvil-memory-summaries=
- topic ごとに集約、関連 observations へ drill-down

** Phase 6 — Promote to auto-memory (SHIPPED 2026-04-25, merge 4f5814a)
*** 6.1 候補抽出 (cron / 手動)
- 週 1 回 =M-x anvil-memory-promote-candidates=
- 高 importance + 同 topic 反復 + auto-memory 既存と非重複の summary を提示
*** 6.2 1-click promote
- =memory-save-check= 既存 tool に対象 summary を渡す → frontmatter 整形 →
  MEMORY.md 追記
*** 6.3 双方向は意図的に **NG**
- auto-memory → observation 書き戻しはしない (人手 curated 層を尊重)

** Phase 7 — Vector embedding (optional, deferred)
- =anvil-http= (Doc 09) 経由で OpenAI / local embedding API 呼ぶ
- summaries に vector column 追加、cosine sim search
- 必要性は Phase 1-6 運用後に判断 (FTS5 trigram で十分なケースが多い見込み)
