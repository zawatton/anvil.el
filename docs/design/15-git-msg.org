Tier 2 of Week-2 roadmap.  Extends anvil-git with commit-message and
PR-body generators, plus enriched blame.  All changes Claude-side are
covered by memory =feedback_commit_co_authored_by.md= (Co-Authored-By
trailer mandatory).

* Motivation

Every phase ship in this project generates:

1. 1 commit message ("feat(X): ...") with structured bullets.
2. 1 merge message (~Merge feat/X into develop~).
3. 1 MEMO entry in journal.
4. Later, 1 PR description if self-PR cycle is used.

All four get derived from the same diff + commit-log data.  Right now
Claude hand-writes each and the user pastes.  A generator tool that
takes diff/log and emits structured Conventional Commits / PR body
saves ~200–400 tokens per ship + eliminates the Co-Authored-By
forget-to-add class of bugs.

* Goals

1. ~git-commit-message :staged t~ → Conventional Commits draft from
   staged diff.  Auto-appends Co-Authored-By per memory rule.
2. ~git-pr-body :base BRANCH~ → summary + test-plan checklist from
   commit log.
3. ~git-why LINE FILE~ → blame + originating commit subject + diff
   hunk (like git-blame but with context).
4. ~git-commit-split PATH~ → suggest hunks to split a large diff into
   logical commits (preview only).

* Non-goals

- AI-driven commit message generation via orchestrator.  The tool
  is deterministic; Claude writes the message using the structured
  data we provide.  (Users can of course pipe this into orchestrator
  if they want.)
- PR creation (that's gh CLI or mcp__github).  We only produce the
  body text.
- Cross-repo blame / submodules — anvil repo is the target.

* Architecture

All four tools shell out to git via anvil-git helpers (Doc already
shipped), not parse git output raw.  Extract:

#+begin_src elisp
(anvil-git-msg-commit &key staged scope)
  ;; Returns: (:type STR :scope STR-OR-NIL :subject STR :body STR
  ;;           :trailers ALIST :suggested-full STR)

(anvil-git-msg-pr &key base head)
  ;; Returns: (:title STR :summary STR :test-plan LIST :commits LIST
  ;;           :suggested-body STR)

(anvil-git-msg-why LINE FILE &key revision)
  ;; Returns: (:sha STR :author STR :date STR :subject STR
  ;;           :hunk-head STR :co-authors LIST)

(anvil-git-msg-split-suggest PATH)
  ;; Returns: (:hunks ((:group N :files LIST :lines N :summary STR) ...))
#+end_src

* Phases

** Phase 1 — commit-message + pr-body (~SHIPPED~, 2026-04-20)

Deterministic draft generator — no LLM call.  Reads the staged diff
via =git diff --cached --name-status= + =--numstat= (already wrapped
by =anvil-git=), classifies each file into added / modified / etc.,
and:

- Type: all-tests → =test=, all-docs → =docs=, any newly added file
  → =feat=, otherwise =fix=.  Deliberately conservative — users who
  know they wrote a =refactor= / =chore= override the inferred type.
- Scope: each file maps to a module name via
  =anvil-git-msg--module-name-of= (strips the =anvil-= prefix for
  elisp files, special-cases =docs/=, =.github/=, =benchmarks/=,
  =pty-broker/=).  Multiple modules collapse to the longest common
  hyphen-delimited prefix (=orchestrator-routing= and =orchestrator=
  → =orchestrator=).  Mixed modules with no common prefix leave scope
  nil (header prints =type: subject= without parens).
- Subject: the module does NOT guess — it emits the placeholder
  =<describe>= and expects the caller to swap in a human summary.
- Trailers: =Co-Authored-By= is appended by default (value lives in
  =anvil-git-msg-co-authored-by=, settable to nil).
- Output includes =:suggested-full= — the header + body + trailer
  string ready for =git commit -F -= after the subject swap.

PR-body generator reads =git log BASE..HEAD= (up to
=anvil-git-msg-pr-max-commits= = 40), builds a =## Summary=,
=## Commits=, =## Test Plan= markdown body.  Test-plan bullets are
derived from the Conventional-Commits types seen in the range (feat
→ "exercise new commands", fix → "reproduce the bug", etc.).

** Phase 2 — blame + split suggestions

~git-why~: wraps ~git blame -L~ + ~git show~ head.  ~git-commit-split~:
heuristic hunk clustering by file group.  ERT +5.

* Testing strategy

- Fixture repo per test, controlled history.
- Assert trailer presence.  Assert scope choice against golden table.

* Platform notes

- Windows git output has CRLF — force LF in git config
  ~core.autocrlf=false~ for anvil (already set).
- Blame across rename chains: use ~-M -C~ flags.

* Primitive vs Convention

将来 anvil を外部公開する際、以下の境界で分離する:

- *Primitive (GENERIC 化可能)* — staged diff + recent commits からの commit message
  合成、PR body synthesis、blame/why 合成、hunk clustering。汎用 git workflow に
  価値がある。
- *Convention (運用者固有)* — Co-Authored-By Claude の強制付与、journal-YYYY.org
  への MEMO エントリ自動生成、zawatton repo の master 直 push / feat branch +
  self-PR 慣習、日本語 commit message 想定。公開時は optional hook としてオフ既定、
  template は =anvil-git-msg-templates/= にユーザ提供分離。
